Dados e sincronização

Modo online Firebase e sincronização disponíveis

Use os atalhos abaixo para backup, importar JSON e enviar dados para a nuvem. O modo offline/online (chave sot_offline_mode) é alterado nas abas Cadastro de Saídas, Saídas Administrativas e Saídas de Ambulâncias. A legenda indica se a ação altera apenas este navegador ou também o Firebase.

Legenda: Só local — sem escrita na nuvem Local + Firebase (O backup pode ler a nuvem para completar o JSON, mas não a altera.)

Desmarcado: aplica apenas neste navegador (localStorage e IndexedDB da VTR Oficina), sem alterar a nuvem.
Marcado (modo online): após aplicar localmente, pede confirmação e funde com o Firebase (nuvem prevalece no mesmo identificador de registo). Em modo offline a caixa fica indisponível.

Formatos JSON aceites e guia offline
  • Carregar Dados — ficheiro .json com um objeto na raiz (o mesmo tipo que o Backup gera, ex. backup_sistema.json). Substitui o armazenamento local e a VTR Oficina no IndexedDB. A opção «Sincronizar com Firebase após importar» controla se, em seguida, o sistema funde com a nuvem (com confirmação extra quando online); desmarcada, só o navegador é alterado.
  • Restaurar saídas do backup — .json com pelo menos uma das chaves saidasAdministrativas ou saidasAmbulancias. Cada valor pode ser um array ou uma string que o JSON.parse interprete como array. Pode usar o ficheiro de Baixar saídas (JSON) ou um excerto de backup completo que contenha essas chaves.
  • Baixar saídas (JSON) — gera SOT_saidas_DD-MM-AAAA.json nesse formato mínimo, ideal para outro PC ou para trabalhar sem internet (dados ficam no navegador até enviar à nuvem).

Sem internet: use Restaurar saídas com o JSON preparado; em seguida Enviar Dados para Nuvem (ou Sincronizar) quando voltar a estar online e autenticado, para atualizar o Firebase.

Abrir guia EXPORTAR_SAIDAS_OFFLINE.md Documento na pasta do projeto, junto a esta página (mesmo nível que Configurações). Se o browser não abrir, abra o ficheiro no explorador ou num editor.
